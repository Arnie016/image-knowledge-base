# Image Atlas Curation Rubric

This rubric is the public platform filter for deciding what gets surfaced first and what gets archived from the default catalog view.

## Scale

Every image receives a `1-5` score on five dimensions, then a weighted total out of `100`.

- `Specificity` — concrete nouns, precise world logic, disciplined scale, and non-generic framing.
- `Usefulness` — how reusable the image is for searchers, teaching, product briefing, editing, or prompt remixing.
- `Vibe` — strength of mood, palette, material language, and atmospheric cohesion.
- `Search Fit` — how many hyper-specific hooks the image exposes for search and filtering.
- `Novelty` — whether the image expands the corpus instead of repeating an existing branch.

## Bands

- `Signature` (`90-100`) — front-rank images that are highly specific, strong in vibe, and broadly useful.
- `Strong` (`80-89`) — high-value images worth surfacing early in search and browsing.
- `Useful` (`70-79`) — supporting images that add real breadth or documentation value.
- `Niche` (`0-69`) — narrower images that stay available but should not dominate discovery.

## Archive Rule

Archiving is manual and conservative. Images are archived from the default platform view only when one of these is true:

- an older run is clearly superseded by a stronger later run
- the branch is redundant enough to hurt discovery quality
- a human audit marks the image or run as not needed for public browsing

Archived images remain on disk and in the data model, but the catalog hides them by default unless `Show archived` is enabled.
